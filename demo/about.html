How it works

Three steps on every navigation.

1. Intercept the click

A single delegated listener on document catches all link clicks. External links, target="_blank", modifier keys, and anything with fiamma-off are passed through normally.

2. Fetch and parse

The target URL is fetched via XHR. The raw HTML is parsed into a detached Document using DOMParser. Non-200 responses fall back to a normal browser navigation.

3. Swap the DOM

The <head> is reconciled — stale nodes removed, new ones appended. The <body> is replaced wholesale. Body scripts re-run. Scroll resets (or jumps to a hash target). History is pushed.

Preserving elements

Add fiamma-preserve and a unique id to any element. Before the body swap, Fiamma plucks those nodes out of the incoming document and replaces them with the live ones already on the page.

The navbar on this page has id="navbar" fiamma-preserve. It has never been destroyed or recreated — you can attach event listeners, store state, run animations, and they all survive navigation. 
<nav id="navbar" fiamma-preserve>
  <!-- lives forever -->
</nav>

Events

Hook into the lifecycle from anywhere:

window.addEventListener('fiamma:fetch', () => { /* fetch started */ });
window.addEventListener('fiamma:end',   () => { /* DOM swapped   */ });
